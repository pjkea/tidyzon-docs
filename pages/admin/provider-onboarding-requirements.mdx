## Notes

- All four endpoints are served by the `getSubmissionsAdmin` lambda, which dispatches based on the route suffix.
- `results[]` is an empty array (not null) when a provider hasn't submitted anything yet.
- The `requirements` endpoint response is keyed by mode name (a dynamic object), not a fixed array — iterate over `Object.entries(results)` in the frontend.
