## Notes

- Only payments with `status = 2` (completed) are included in statement totals. Pending and failed payments are excluded.
- `statementId` is a composite string key: `{type}-{year}-{period_number:02d}`. Use this ID to fetch or act on a specific period.
- **⚠ Known bug**: `GET /v1/admin/statements` currently returns HTTP 500 when no completed payments exist.

## Notes

- All five endpoints are served by `statementHandlerAdmin`.
- **ID inconsistency**: `GET /v1/admin/statements/{id}` uses the **composite string ID** (`monthly-2026-04`), while `mark-paid` and `mark-pending` use the **numeric payment ID** (`paymentid`).
- `pdfAvailable` is always `false` — PDF statement export has not been implemented.
- Only payments with `status = 2` (completed) appear in `GET /v1/admin/statements` — newly created payments with `status = pending` will not appear in the list until marked paid.
