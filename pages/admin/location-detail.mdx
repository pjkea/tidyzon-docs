## Notes

- This endpoint is a lookup by **row ID**, not by provider/user ID. Use `GET /v1/admin/locations/providers/{tidyspid}` or `GET /v1/admin/users/{userid}/locations` for entity-specific lookups.
- The response has **no message wrapper** — the object is returned directly.
- Provider-only fields (`verified`, `rating`, `completedJobs`, `lastPing`, `serviceType`) are omitted from user responses.

## Notes

- All three endpoints are served by `locationHandlerAdmin`, which dispatches based on path structure.
- The response has **no message wrapper** — the data object is returned directly.
- `GET /v1/admin/users/{userid}/locations` returns **multiple entries** — one per row in `service_location_history`. The `id` field repeats the user ID (not a row ID).
- `status` for users uses a 15-minute activity window: `online` if `last_app_opened_at >= NOW() - 15 min`, otherwise `offline`.
