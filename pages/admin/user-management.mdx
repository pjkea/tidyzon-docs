      }
    },
    { status: 400, description: 'No fields provided, or invalid roleid.', sample: { message: "At least one field must be provided for update" } },

  ]}
/>

## Notes

- At least one field must be present in the body — sending an empty object returns 400.
- `changes` reflects only the fields you sent; fields you did not include are not modified.
- If no `userdetails` row exists for the user yet, the endpoint creates one (INSERT) rather than failing.
- `roleid` is validated against the roles table before updating; an invalid role returns 400.
