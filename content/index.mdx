# Quickstart Guide

Tidyzon's APIs follow **REST principles** and accept requests and return responses in **JSON**. All APIs are available over HTTPS.

**Base URL:** `https://25p5ndzk5j.execute-api.us-east-1.amazonaws.com/dev`

## Requests and Responses

The request body data and response data are structured in JSON format. All responses follow this envelope:

```json
{
  "message": "Success",
  "data": { ... }
}
```

| Field | Type | Description |
|-------|------|-------------|
| `message` | string | Human-readable description of the result |
| `data` | object \| array | The payload returned by the request |

---

## 🔑 Authentication

Most endpoints require a JWT Bearer token in the `Authorization` header:

```
Authorization: Bearer <id_token>
```

There are three auth levels:

| Level | How to obtain |
|-------|--------------|
| **Admin** | `POST /v1/admin/auth/login` with `{email, password}` |
| **User / Customer** | `POST /v1/auth/sign-in` with `{identifier, password}` |
| **Public** | No token required |
