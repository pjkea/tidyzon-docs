## Notes

- `GET /v1/admin/transactions/{transactionid}` accepts both a numeric `paymentid` and a string `transactionid` reference — the query matches on `WHERE paymentid = %s OR transactionid = %s`.
- `payout` and `status` return the same value; `payout` exists for dashboard UI compatibility.
- `tidyzonCharge` is `null` when no `request_pricing` row with `is_summary=true` is linked to the order.
- The `POST` endpoint uses the payments table — not a separate transactions table. "Transaction" in this context refers to a payment entry.
