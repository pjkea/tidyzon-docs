## Notes

- This list is used by both the **provider onboarding flow** (to know which documents to request) and the **admin review panel** (to label each uploaded file).
- `is_required: false` documents are optional but displayed — providers may still upload them for faster verification.
- `is_active: false` types are legacy entries; they are not shown in the upload UI but may appear on old provider records.
- `displayorder` controls the sort order in both the provider app and the admin checklist — lower numbers appear first.
- The sibling endpoint `GET /v1/admin/providers/onboarding/documents` lists the actual uploaded files for a specific provider.
