## Status Logic

| Status | Condition |
|--------|-----------|
| `active` | `isactive = 1` AND `lastloginat >= NOW() - 30 days` |
| `dormant` | `isactive = 1` AND (`lastloginat IS NULL` OR `lastloginat < NOW() - 30 days`) |
| `inactive` | `isactive = 0` |

## Notes

- `avg_rating` is the literal string `"new"` (not `0`) when the user has no ratings. The frontend coerces `"new"` → `0` before rendering a star widget.
- `dormant` API status is labelled **"Deactivated"** in the dashboard UI (danger/red badge). This is intentional — both `dormant` and `inactive` map to red badges.
- The dashboard "TOTAL NUMBER OF USERS" card calls `?page=1&limit=1` and reads only `pagination.total`.
- `since` and `last_active` are pre-formatted server-side as `%B %d, %Y %I:%M %p`. The UI does not re-format them.
