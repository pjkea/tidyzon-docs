## Status IDs

| ID | Name | Meaning |
|----|------|---------|
| 1 | pending | Submission received, not yet reviewed |
| 2 | under_review | Admin has opened the review |
| 3 | approved | Accepted — item counts toward onboarding progress |
| 4 | rejected | Rejected — provider must re-submit |

## Notes

- `reference_id` comes from the `submission_id` in the documents list (`GET /v1/admin/providers/onboarding/{tidyspid}/documents`) or the `requirement_item_id` from the requirements list.
- The action is logged to the transaction table with `transactiontypeid = TRANSACTION_PROFILE_UPDATED`.
- Approving all required documents + requirements automatically unlocks the provider for activation.
