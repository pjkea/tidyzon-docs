## Notes

- All routes are served by the `notificationHandlerAdmin` lambda which dispatches on method + path suffix.
- The typical workflow: `POST /notifications` → `POST /notifications/{id}/send` (immediate) or `POST /notifications/{id}/schedule` (future).
- `deliveryStats.failed` counts FCM tokens that were invalid or unreachable — not an error in the API call itself.
