## Notes

- When `providerid` is passed, switches to an orders-based query and also returns orphaned requests (provider was assigned but rejected/timed out with no order row).
- `datetime` is resolved to the request's local timezone using `timezonefinder_from_coordinates`. Falls back to `created_at` when `scheduleddatetime` is null (live requests).
- `rating` is the customer's rating of the **provider** (`recipient_side = 'provider'`). The entire object is `null` when no rating has been submitted.
- The **REQUEST STATISTICS** stat boxes (Completed / Cancelled / Number Of Requests) are computed **client-side** from the returned `requests[]` array — NOT from `data.statistics`.
