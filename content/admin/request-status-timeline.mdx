## Reason Codes

| Code | Actor | Meaning |
|------|-------|---------|
| `T1` | Provider | Provider sent offer (normal) |
| `T2` | Provider | Provider modified offer |
| `T3` | Provider | Provider cancelled offer |
| `R1` | Customer | Customer accepted offer |
| `R2` | Customer | Customer rejected offer |
| `R3` | Customer | Customer cancelled request |
| `A1` | Admin | Admin manually advanced status |
| `A2` | Admin | Admin force-cancelled |
| `A3` | System | System timeout / auto-cancel |

## Notes

- The list endpoint (`/status-timeline`) is useful for cross-request audit trails and date-range exports.
- The per-request endpoint (`/{requestid}/timeline`) is what the Manage Request modal loads — it gives the full ordered story for one request.
- `metadata` is only populated on offer rows (status = `offer_sent` or `offer_modified`). It contains `price` and the provider snapshot at the time of the offer.
- `changedby_userid` is `null` for system-initiated transitions (timeouts, SNS-triggered auto-completions).
